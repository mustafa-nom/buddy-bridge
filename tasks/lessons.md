# Lessons

This file tracks corrections and patterns to avoid repeating.

## Current Rules

- Do not make the game feel like a quiz.
- Do not add long educational text.
- Every lesson must also be a fun mechanic.
- Runner and Guide must have different information.
- Guide must have meaningful actions, not just reading.
- Keep scope small for hackathon.
- Build 3 polished rooms before adding optional content.
- Keep files under 500 lines.
- Parameterize reusable functions instead of hardcoding one-off behavior.
- Avoid hardcoded helper functions that break future room variants.
- Server owns gameplay truth.
- Client is display and input only.
- If a module sets up state or listeners, make sure bootstrap requires it.
